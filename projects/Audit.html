Audit Management



Online portal that lets authorized users to select b/w yes or no responses for Internal/SOX Audit Type questions and reflecting the same on result page based on audit type response captured.


Source Code

Case Study

This project was assigned to us as a team to come up with a system to automate the Audit processing, to make the management scalable and ensure clarity and ease of tracking. Hence we had to come up with a Solution with help of Full Stack. After going through the problem statement and functionality we discussed and divided work among ourselves.

Audit Management Portal allows user to login. Once successfully logged in, the following operations needs to be done: Choose the audit type to view the list of audit checklist questions wherein the project manager provides answers to the set of questions. Once it is done, the Audit Severity Microservice is invoked which determines the project execution status after analyzing the same, the result will be displayed on the portal. The audit response details along with the project execution status and remedial action duration are saved in the database.


This Project was divided into 4 Modules for Backend :


  1. Authorization Microservice
  2. Audit Checklist Microservice
  3. Audit BenchMark Microservice
  4. Audit Severity Microservice

Portal Summary

In order to enter the Portal, the project Manager has to login using their Credentials (Authorization Microservice)and then the Project Manager gets an option to choose between the Internal/SOX Type Audit Questions.Based on the Option selected by the project Manager, the Audit Questions will be displayed which will be Yes/No type questions.(Audit Checklist Microservice) and the options are selected.

The response will be captured and results are processed with the help of "Audit benchmark Microservice" Then the result sheet will be displayed to the Project manager along with the actions to be taken based on the response.
